Does Relative Size Units Matter to Create a Responsive Element?

Yes. Since the only other way is to use a unit like px, one would be forced to zoom the page using the browser's zoom page capability to increase the apparent font-sizes, but that may cause havoc to general structure of the page or even possibly make scrolbars, including horizontal, necessary to view the whole content.

For a relative size unit on the other hand, one only needs to change the browser's default font-size. Since the content would be relative, it would scale up or down, depending on one's chosen value, seamlessly without problems like using px would cause. So, by thier nature relative sizes are a good tool for creating a responsive element.

How does the browser interpret 200px if 16px is it's default?

If 16px is the browser's defualt, it means that the browser uses 16px as the unit reference to scale other size values. It would scale the font-size of elements by comparing thier given size, for example 200px, to 16px, in this case making the element 200/16 (= 12.5) of the default size.

Some New Additions to CSS3:

Rounded corner: This addition allows the creation of rounded corners for the border's edge of any element. Here is the shorthand property it uses: border-radius. It sets the radius of curvature for all sides.

Shadow: This is one of a simple looking addition that greatly changes the look and feel of web pages. It gives shadows to elements, giving them life and realisiticity to the eye. Box-shadow and text-shadow are two important types of shadow elements. The former applies to elements (not texts/characters) while the latter applies to texts.

New background properties: One of the new things added to the background property is background-image. Throught this property you can add more than one background images. Background-size is another new background property that allows you to give a specific size for the background images. It has a value called cover that allows you to fill the whole page with an image.

Colors: Besides the normal color name, hexadecimal and RGB values for color properties, CSS3 introduces RGBA, HSL, HSLA and opacity. RGBA is nothing but an RGB plus a parameter that indicated the transparency of the RGB color. Opacity is this parameter standing as a one property.

Gradient: This allows you to display a smooth transition of colors. There are two types: linear and radial. Linear gradient has a specific direction (up/down/right/left/diagonal) to which the color gradually changes, while radial gradient, as its name imply, has radial lines of direction of color change.

Text-overflow: This is a very handy CSS3 tool that lets you specify how to display overflowed content. Its options include clipping and replacing with ellipsis.

Transform: These property allows you to make 2D/3D transformation on elements positioned by the box model (has a display: block) using simple propery-value syntax where the values are mostly transform-functions().

Flexbox and Media Queries: Flexibox is an amazing layout managment tool in CSS3 that is makes page layout accomodate different devices and screens. It solves problems like vertically centering a block inside its parent and making all columns in a multiple column layout adopt the same height even if they vary in content size. The new CSS3 media queries build on the previous CSS3 media types, and they look at the capabality of a device; not its type.
